# Фаза 4: Широкополосный CW Skimmer (План Архитектуры)

## Обзор и Цели
Задача: реализовать многоканальный телеграфный (CW / Morse) декодер-скиммер, который анализирует всю звуковую полосу пропускания (0 - 3000 Гц) одновременно, автоматически находит активные телеграфные сигналы и декодирует их параллельно.

Вместо классического бегущего водопада будет реализован специализированный **Вертикальный Радар (Vertical Spectrum)**. Ось частот располагается по вертикали (как в профессиональных SDR-программах вроде CW Skimmer или SDRuno), а декодированный текст появляется прямо напротив пика соответствующей станции.

## 1. Архитектура Цифровой Обработки (DSP)

Поскольку нам нужно одновременно слушать множество станций, классический подход "один фильтр на весь процессор" не подходит. Архитектура RP2350 (два ядра по 150 МГц) позволяет реализовать гибридный **FFT + Банк Фильтров** подход:

### Элемент 1: Широкополосный поиск (Core 1)
- Ядро 1 берет на себя роль "Скиммера" (наблюдателя). Оно непрерывно получает массивы данных от Ядра 0 и выполняет большое БПФ (FFT на 1024 или 2048 точек) для получения высокого частотного разрешения (~5-9 Гц на бин).
- Выполняется алгоритм поиска пиков (Peak Detection) и оценки отношения сигнал/шум (SNR) для каждого пика.
- Ядро 1 формирует динамический массив "Активных частот" (например, до 10-15 самых громких станций) и передает его Ядру 0.

### Элемент 2: Многоканальный Демодулятор (Core 0)
- Ядро 0 продолжает молотить сэмплы с АЦП на частоте 10 кГц (строгий тайминг).
- Вместо одного демодулятора, Ядро 0 динамически создает **Банк Фильтров (Filter Bank)**. На каждую активную частоту из списка, переданного Ядром 1, назначается независимый узкополосный демодулятор (например, оптимизированный фильтр Гёрцеля (Goertzel) или цифровой понижающий конвертер (DDC) с CIC-фильтром).
- Математика RP2350 легко потянет 10-15 параллельных каналов DDC.

### Элемент 3: Адаптивный CW Декодер
Каждый из 15 каналов имеет собственный независимый автомат состояний (Framer) для морзянки:
1. **Envelope Detector:** Определение наличия тона (Нажатие / Отжатие).
2. **Timing Estimator:** Автоматическая подстройка скорости (WPM). Алгоритм собирает статистику длительностей нажатий, разделяя их на "Точки" (Dots) и "Тире" (Dashes), а паузы на межсимвольные и межсловесные (Farnsworth timing).
3. **Decider:** Перевод точек и тире в бинарное дерево Морзе и выдача ASCII символов.

## 2. Концепция Пользовательского Интерфейса (UI)

Экран будет поделен на две логические зоны (Вертикальный радар):

*   **Левая колонка (20-30% экрана): Вертикальный Спектр.**
    *   Частота 0 Гц внизу, 3000 Гц наверху.
    *   Рисуется график спектра (пики выпирают вправо).
    *   Это позволит визуально оценить загруженность диапазона.
*   **Правая колонка (70-80% экрана): Текстовая сетка.**
    *   Как только Ядро 0 декодирует букву на частоте `X` Гц, эта буква печатается в правой зоне, причем строка текста находится **на той же высоте `Y`**, что и пик частоты `X` на спектре слева!
    *   В результате мы получаем самописную "стену" текста. Каждая строка — это отдельная радиостанция. Текст ползет слева направо.
    *   Станции с разной частотой будут печатать текст на разных строках.

## 3. Вызовы и Сложности
1. **Ложные срабатывания (QRM/QRN):** Треск статики может восприниматься как точки. Потребуется математика подавления импульсных помех (Noise Blanker) до входа в банк фильтров.
2. **Плавающие сигналы (Drift):** Сигналы станций (особенно старых) плывут по частоте. Широкополосный FFT должен отслеживать дрейф пика и "подруливать" частоту узкополосного фильтра Ядра 0 (по сути, 15 независимых AFC).
3. **Ресурсы дисплея:** Рисование до 15 обновляемых строк текста одновременно может перегрузить SPI-шину дисплея. Потребуется умный алгоритм отрисовки (Dirty Rectangles / Text buffering), чтобы отрисовывать только те буквы, которые добавились в эту долю секунды.

## 4. План действий (Roadmap)
1. **Шаг 1:** Реализовать один идеальный CW-декодер (Goertzel + Auto-WPM) на Ядре 0 с ручной настройкой на частоту, чтобы отточить алгоритм распознавания морзянки.
2. **Шаг 2:** Разработать класс `FilterBank`, который сможет динамически добавлять/удалять частоты "на лету".
3. **Шаг 3:** Реализовать Вертикальный UI и алгоритм поиска пиков на Ядре 1.
4. **Шаг 4:** Интегрировать связку "Скиммер нашел -> Передал Ядру 0 -> Ядро 0 декодирует -> UI рисует строку".
