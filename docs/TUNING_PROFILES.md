# Tuning Profiles: Оптимальные параметры DSP

Этот документ фиксирует найденные оптимальные параметры для различных типов сигналов.
Параметры получены автоматическим тюнингом (`tools/autotune.py`) на реальных эфирных сигналах.

## Параметры DSP

| Параметр | Описание | Диапазон |
|----------|----------|----------|
| ALPHA | Ширина петли DPLL (PI-регулятор). Больше = быстрее захват, но шумнее | 0.005 — 0.200 |
| BW | Ширина Biquad LPF (K). Меньше = чище, но чувствительнее к drift | 0.30 — 2.00 |
| SQ | Порог шумоподавителя по SNR (dB). Больше = строже | 1.0 — 20.0 |

## Profile 1: NAVTEX 4583 kHz (50 Baud, 170 Hz Shift)

**Дата:** 2026-04-04 (ретюнинг)
**Источник:** WebSDR → 4583 kHz USB → audio cable → ADC
**Протокол:** B 50, S 170, ST 1.5

| Параметр | Default | Optimized | Комментарий |
|----------|---------|-----------|-------------|
| ALPHA | 0.0350 | **0.0650** | Умеренно шире петли — быстрая адаптация к дрейфу |
| BW | 0.75 | **1.30** | Шире фильтр — сигнал относительно чистый на 4583 |
| SQ | 6.0 | **8.0** | Жёстче порог — отсекает мусор в паузах |

**Результат:** ERR 8% → 6%, SNR ~60 dB, DPLL freq error < 0.0001, Score 27→42

## Методология

Автотюнинг выполняется скриптом `tools/autotune.py`:
1. Подключение к устройству через USB Serial
2. Включение диагностического потока (`DIAG ON`)
3. Sweep каждого параметра по всему диапазону (Phase 1)
4. Fine-tuning лучших значений с половинным шагом (Phase 2)
5. Итоговый замер и сохранение (`SAVE`)

Метрика качества: `Score = -5*ERR + SNR - 1000*|DPLL_FE| + SQ_bonus`

Для максимальной точности рекомендуется:
- Стабильный сигнал на протяжении всего тюнинга (~4 минуты)
- Увеличить время замера (без `--quick` = 6 сек на точку)
- Повторить тюнинг 2-3 раза для проверки воспроизводимости
