# Фаза 3: Профессиональный DSP Демодулятор RTTY (Итоги)

Этот документ описывает финальную архитектуру и решения, примененные при создании программного RTTY демодулятора уровня fldigi на микроконтроллере RP2350.

## 1. Архитектура конвейера (Core 0)
Весь DSP-конвейер работает на Ядре 0 с жестким таймингом ровно 10 000 итераций в секунду (10 кГц). Для предотвращения фазового джиттера (jitter), расчеты FFT и отрисовка графики вынесены на Ядро 1.

1. **АЦП (ADC):** Считывание сэмплов с разрешением 12 бит.
2. **FIR Bandpass Filter:** 63-taps полосовой фильтр для предварительной очистки сигнала.
3. **I/Q Demodulator:** Входящий сигнал умножается на локально генерируемые синусы и косинусы частот Mark и Space, переводя полезный сигнал на нулевую частоту (baseband).
4. **Biquad LPF (Extended Raised Cosine):** Фильтрация I и Q ветвей для каждого тона. Коэффициент ширины фильтра `K = 0.75` по умолчанию.
5. **ATC (Automatic Threshold Correction):** Детекторы огибающей с быстрой атакой и медленным спадом (FASR) независимо отслеживают замирания (fading) тонов Mark и Space. Это позволяет динамически корректировать порог принятия решения (Дискриминатор D).
6. **DPLL (Digital Phase-Locked Loop):** ПИ-регулятор (Пропорционально-Интегральный) для синхронизации битов.
7. **Baudot Framer:** Конечный автомат, собирающий биты в 5-битные посылки ITA2 и декодирующий их в ASCII с учетом регистров LTRS/FIGS.

## 2. Критические проблемы и их решения

### 2.1. Проблема 1.0 стоп-бита (Continuous Streaming)
**Симптомы:** На скорости 75 Бод с 1.0 стоп-битом прием сплошного текста (RYRYRY) постоянно срывался в ошибку кадра (`[ERR]`), в то время как 1.5 стоп-бита принимались идеально.
**Причина:** При 1.0 стоп-бите следующий стартовый бит начинается мгновенно после окончания стоп-бита. Изначально конечный автомат (Framer) сбрасывал фазу DPLL на границе символа. Из-за дрейфа тактовых генераторов (между ПК и Pico) фаза "уплывала", и декодер пропускал мгновенный спад в Space.
**Решение:** Внедрена логика Continuous DPLL. При ожидании 1.0 стоп-бита автомат проверяет полярность сигнала сразу после чтения стоп-бита. Если сигнал уже отрицательный (Space), автомат *мгновенно* переходит в состояние приема стартового бита без ожидания перепада (zero-crossing), принудительно устанавливая `symbol_phase = 0.0f`.

### 2.2. Дрейф генератора (DPLL Integral Windup)
**Симптомы:** Прием ломался через несколько секунд после начала передачи.
**Причина:** Разная реальная скорость передачи (например, 74.8 Бод вместо 75.0 Бод). Пропорциональная подстройка фазы (`symbol_phase -= alpha * err`) не справлялась со статической ошибкой частоты.
**Решение:** Добавлен интегральный коэффициент `freq_error` (BETA = ALPHA²/2). DPLL теперь работает как полноценный ПИ-регулятор, подстраивая внутренний таймер под реальную скорость источника. Ограничитель `max_fe = 0.05f * phase_inc` предотвращает интегральное насыщение (windup) при отсутствии сигнала.

### 2.3. Squelch (Шумоподавитель) и Замирания (Selective Fading)
**Симптомы:** Декодер глотал случайные буквы при небольших просадках громкости сигнала.
**Причина:** Жесткий порог отключения по абсолютному уровню сигнала и SNR.
**Решение:** Внедрен логарифмический расчет SNR (в дБ) и гистерезис.
* Открытие Squelch: Сигнал > -60 dB, SNR > 4.0 dB.
* Закрытие Squelch: Сигнал < -65 dB или SNR < (SQ_SNR - 2.0) dB.
Это позволяет декодеру продолжать тянуть слабый сигнал на границе шумов во время эфирных замираний. В будущих фазах планируется добавить регулировку порога гистерезиса в меню настроек.

### 2.4. Режим LSB и Инверсия
**Симптомы:** Для приема сигнала приходилось нажимать `INV` даже для стандартных посылок.
**Причина:** Ошибка в начальной математике демодулятора: Mark считался верхней частотой, а Space - нижней (стандарт USB/AFSK).
**Решение:** Алгоритм строго приведен к стандарту радиоэфира LSB (Lower Sideband) и FSK:
* **Mark (M)** = Нижняя звуковая частота (`Center - Shift/2`).
* **Space (S)** = Верхняя звуковая частота (`Center + Shift/2`).

### 2.5. Красный водопад (ILI9488 DMA Quirk)
**Симптомы:** Вместо синего фона рисовался красный.
**Причина:** SPI дисплей ILI9488 аппаратно залочен в Mode 11 (BGR out, 16-bit swapped) при использовании 18-битной панели по 16-битной шине DMA.
**Решение:** Вместо программного swap-а в цикле `color565`, аргументы `(r, g, b)` передаются в функцию цвета в обратном порядке: `color565(b, g, r)`. А перед отправкой в DMA делается аппаратный `swap` байтов `(c >> 8) | (c << 8)`.

## 3. Пользовательский Интерфейс и UX (Новое в ветке оптимизации)

### 3.1. Подменю Диагностики (DIAG)
Добавлен отдельный диагностический экран с расширенным функционалом:
*   **Rainbow Palette:** Наглядный ориентир для оценки цветового маппинга водопада.
*   **Zero Bias Meter:** Индикатор для аппаратной подстройки смещения АЦП (1.65В) с цветовой индикацией точности.
*   **Управление шириной строки (LINE WIDTH):** Динамическая подстройка текста (30-85 символов).
*   **Serial Diag Toggle:** Включение/выключение отладки в USB-порт.

### 3.2. Умный перенос строк (Smart Newline)
Реализован алгоритм, схлопывающий повторы `CR/LF` и `CR CR LF`, характерные для механических телетайпов, при этом сохраняя пустые строки для структуры документа.

### 3.3. Оптимизированная система шрифтов (Build 188)
В ходе финальной полировки интерфейса количество шрифтов было сокращено до двух наиболее эффективных режимов:
*   **F2:NORM (Standard):** Классический шрифт 8x16 пикселей. Установлен межстрочный интервал **17 px**, что делает текст более слитным и приятным для чтения.
*   **F0:NARW (Narrow):** Компактный шрифт 6x8 пикселей. Убран дробный масштаб, теперь это чистый попиксельный вывод с интервалом **10 px**. Позволяет выводить до 16 строк текста (идеально для DWD SYNOP).

## 4. Консоль настройки и Сохранение (Tuning & Flash)
Через USB Serial (9600 бод) доступен интерфейс реального времени. Команды: `ALPHA`, `K`, `SQ`, `CLEAR`.
Все настройки (Скорость, Сдвиг, Стоп-биты, Инверсия, Порог фильтров, Шумодав, Режим отображения) автоматически сохраняются во внутреннюю Flash-память спустя 15 секунд после последнего изменения.

**Сброс калибровки и настроек (Reset):**
1. **Экран (Touch Reset):** Зажмите экран при включении питания.
2. **Валкодер (Encoder SW):** Зажмите кнопку энкодера (GPIO 4) при включении питания.
